Что такое HTML?

  • HTML - это стандартный язык разметки для создания веб-страниц.
  • HTML обозначает язык разметки гипертекста;
  • HTML описывает структуру веб-страниц с использованием разметки;
  • HTML-элементы являются строительными блоками HTML-страниц;
  • HTML-элементы представлены тегами;
  • HTML-теги обозначают фрагменты контента, такие как «заголовок», «параграф», «таблица», «изображение» и т.д.;
  • Браузеры не отображают теги HTML, но используют их для отображения содержимого страницы.

Веб-браузеры

Задача веб-браузера - считывать HTML-документы и отображать их.

Наиболее популярные браузеры: (Chrome, Firefox, Opera, Safari).

Браузер не отображает теги HTML, он использует их для определения способа отображения документа:

HTML документ обычно состоит из следующих элементов:

  • Объявление <!DOCTYPE html> определяет документ как HTML5;
  • Элемент <html> является корневым элементом HTML-страницы;
  • Элемент <head> содержит метаинформацию о документе;
  • Элемент <title> указывает заголовок документа;
  • Элемент <body> содержит видимое содержимое страницы;
  • Элемент <h1> определяет большой заголовок;
  • Элемент <p> определяет параграф (абзац);

Это минимальный набор элементов для веб-страницы.

HTML-теги

HTML-теги - это имена элементов, окруженные угловыми скобками:

<имя тега>Содержание здесь...</имя тега>

HTML-теги обычно парные, как, например, теги параграфа <p> и </p>.

Первый тег в паре - это начальный тег, второй тег - конечный тег. Конечный тег написан как начальный тег, но с косой чертой (слешем), вставленной перед именем тега.

Начальный тег также называется открывающий тег, а конечный тег - закрывающий тег.

Основные примеры

Простой HTML-документ

Пример 6.1:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="utf-8">
<title>Название страницы</title>
</head>
<body>
<h1>Первый заголовок</h1>
<p>Первый параграф.</p>
</body>
</html>
Посмотреть результат!

Это простой веб-документ. Сохраните его в редакторе и откройте в браузере. Вы увидите, что в браузере отображается только та информация, которая находится в теле документа между тегами <body></body>.

Тип документа <! DOCTYPE>

Любая HTML-страница начинается с объявления <!DOCTYPE> (типа документа). Объявление <!DOCTYPE> позволяет браузерам правильно отображать веб-страницы.

Он должен появляться только один раз, в верхней части страницы (перед любыми тегами HTML).

Объявление <!DOCTYPE> не чувствительно к регистру.

Объявление <!DOCTYPE> в спецификации HTML5 выглядит так:

<!DOCTYPE html>

После определения типа документа в начальном (открывающем) теге <html> необходимо указать язык, на котором преимущественно написано содержание страницы, например, русский (ru).

<html lang="ru">

Это необходимо для поисковых роботов и различных веб-приложений, использующих информацию на данной странице.

Элемент HTML <head>

Элемент HTML <head> (голова или заголовок) не имеет ничего общего с заголовками на странице (тегами h1 - h6). Элемент <head> - это контейнер для метаданных. Метаданные HTML - это данные о документе HTML. Метаданные не отображаются на странице. Элемент <head> помещается между тегом <html> и тегом <body>:

Пример 6.2:

<DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>Моя первая HTML-страница</title>
</head>
<body>
<p>Элемент head содержит метаданные</p>
<p>Метаданные - это данные о документе HTML</p>
</body>
</html>
Посмотреть результат!

Примечание. Метаданные обычно определяют заголовок документа (title), набор символов (кодировка), стили (CSS), ссылки (link), скрипты (JavaScript) и другую метаинформацию.

Во всех предыдущих примерах мы использовали минимальный набор метаданных на веб-странице: <title> (обязательный тег согласно спецификации HTML5), и <meta charset-"utf-8"> (универсальная кодировка, которая используется для распознавания кириллицы).

Раздел head обычно содержит следующие элементы meta (метатеги):

<meta charset="utf-8">
- указывает кодировку страницы utf-8, обычно данный метатег пишется самым первым;
<meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">
- адаптивный метатег для правильного отображения касаний и масштабирования для различных устройств: смартфонов, планшетов и прочих гаджетов с различным разрешением дисплеев;
<meta http-equiv="x-ua-compatible" content="ie=edge">
- метатег управляет режимом отображения страниц в браузерах IE8+. Тег должен находиться в секции head страницы перед всеми остальными элементами, за исключением <title> и других мета-элементов. В противном случае он игнорируется браузерами!

Эти три метатега рекомендуется использовать на всех веб-страницах!

Следующие метатеги являются не обязательными, но желательными и рекомендуются для SEO (продвижения и раскрутки сайта в сети). Хотя жесткого требования их использовать на странице нет! Их отсутствие никак не влияет на валидность веб-страницы и соответствие стандарту HTML5.

<meta name="description" content="описание содержания страницы">
- содержание данного тега может использоваться в сниппетах (описаниях сайтов на странице результатов поиска). Когда-то информация из этого метатега обязательно отображалась в поисковой выдаче и имела особый вес при индексации, однако, в связи со злоупотреблениями, различные поисковые системы используют разные алгоритмы для оценки релевантности указанного в теге описания и содержимого страницы, на основании чего могут игнорировать содержимое тега. Несмотря на это, предпочтительно все же заполнять этот тег, но учитывать, что эффективная длина текста, который можно поместить в этот тег различными поисковиками устанавливается по своему усмотрению и на текущий момент рекомендуемый размер составляет от 70 до 140 символов, а максимально обрабатываемый не превышает 400 символов.
<meta name="keywords" content="ключевые,слова,через,запятую">
- используются слова (обычно до 10), которые встречаются на самой странице. Если в этот тег добавить слова, которые на странице отсутствуют, то такие ключевые слова будут игнорироваться браузерами.

Также можно использовать и другие (менее важные) метатеги, например:

<meta name="copyright" lang="ru" content="Петренко Иван Кузьмич">
- указывает автора или кому принадлежит документ. В настоящий момент имеет крайне низкую актуальность.
<meta name="robots" content="all">
- разрешает индексирование страницы и использование размещённых на ней ссылок для последующей индексации поисковыми роботами.
<meta http-equiv="refresh" content="5; url=http://www.example.com/">
- указывает браузеру автоматически перейти на другой url-адрес через указанное количество секунд. Если параметр url не указан, обновит страницу без использования кэша. Часто недобросовестные веб-разработчики используют данную функцию на некоторых сайтах, чтобы перенаправить пользователей на другой, более нужный ресурс, тем самым накручивая сетевой трафик и увеличивая посещаемость нужного сайта. Поэтому многие поисковики, обнаружив такой метатег на веб-странице, просто игнорируют её, выбрасывая из результатов поиска! Не рекомендуется использовать на сайтах, если хотите, чтобы их видели поисковики!

В связи с тем, что метатеги несут чисто служебную функцию, и, кроме того, значительно увеличивают размер гипертекстового документа, начинающие web-разработчики зачастую игнорируют их, либо используют шаблонные значения, предоставляемые инструментами разработки. Однако грамотное предоставление служебной информации, которую одинаково легко могут обработать и браузеры, и поисковые системы, и стороннее программное обеспечение помогает более успешной индексации страницы в поисковых системах и более тонко управлять поведением браузеров.

Элемент <body>

Информация, которая отображается на странице, находится в теле документа между тегами <body></body>. Семантически основное содержимое страницы обычно заключается в тег <main> (главный, основной). К основному содержимому относятся тексты (заключенные в параграфы), заголовки к текстам, изображения, видео, навигация по странице (сайту) и др. Хотя тег <main> не является обязательным и многие веб-верстальщики его не используют.

Заголовки h1 - h6

Обычно для каждой статьи или раздела используются заголовки, которые определяются тегами от <h1> до <h6>. Тег <h1> определяет наиболее важный заголовок. Тег <h6> определяет наименее важный заголовок:

Пример 6.3:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="utf-8">
<title>Заголовки в HTML</title>
</head>
<body>
<h1>Это заголовок h1</h1>
<h2>Это заголовок h2</h2>
<h3>Это заголовок h3</h3>
<h4>Это заголовок h4</h4>
<h5>Это заголовок h5</h5>
<h6>Это заголовок h6</h6>
</body>
</html>

Результат:

Это заголовок h3

Это заголовок h4

При этом согласно спецификации HTML5 заголовок первого уровня <h1> может использоваться на странице только один раз! Иногда в него заключают название сайта (вместе с логотипом), либо название основной статьи на данной странице. Именно этот заголовок лучше всего ищут поисковые системы (Google, Yandex). Всё, что в него заключено, играет важную роль для нахождения сайта и отображения в результатах поиска. Все остальные заголовки (h2-h6) могут использоваться на странице множество раз.

HTML-параграфы (абзацы) <p>

Обычно любой текст на веб-странице, который пишется с новой строки, заключается в отдельный параграф. HTML-параграфы (или абзацы) определяются тегом <p> (от слова paragraph):

Сразу хочу обратить ваше внимание на то, что в литературе слово paragraph обычно переводится как абзац и на веб-странице отображается, как начало текста с новой строки. Но в среде веб-разработчиков принято больше использовать именно оригинальное название - paragraph, отсуда и тег <p>. Чтобы не было путаницы, я также буду использовать название "параграф", а не абзац.

Пример 6.4:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Параграфы в HTML</title>
</head>
<body>
<p>Это параграф.</p>
<p>Это другой параграф.</p>
</body>
</html>
Посмотреть результат!

Текст на странице может выравниваться по левому краю (по умолчанию), по правому краю, по центру, по ширине. Практически всё также, как в текстовом редакторе Word. Раньше в старом стандарте HTML для выравнивания текста на странице использовали атрибут выравнивания align с необходимым значением (left, right, center, justify). Кроме того, в качестве параграфа можно использовать блочный элемент <div> также как и элемент <p>. Разница между ними в том, что в начале и в конце параграфа (тег <p>) появляется вертикальный отступ (по умолчанию), которого нет при использовании элемента <div>. В разных браузерах этот отступ может отличаться, если вы специально не зададите его в своей каскадной таблице стилей (CSS).

Пример 6.5:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Выравнивание текста в HTML</title>
</head>
<body>
<p align="justify"> Этот текст выравнивается по ширине с помощью значения justify. Но его должно быть много, чтобы увидеть эффект выравнивания. Если текста мало, т.е. если он будет помещаться в одну строку, то выравнивание будет по левому краю.</p>
<div align="right">Этот текст выравнивается по правому краю с помощью значения right.</div>
<p align="center">Этот текст выравнивается по центру с помощью значения center.</p>
<div>Этот текст выравнивается по левому краю (по умолчанию - без указания значения).</div>
</body>
</html>
Посмотреть результат!

Но выравнивание с помощью атрибута align непосредственно в HTML-коде страницы сейчас уже не используется и не соответствует спецификации HTML5. Если вы будете использовать такое выравнивание текста на странице, то ваш HTML-код не пройдёт проверку на валидность. Валидатор будет отображать такие атрибуты как ошибку. Согласно спецификации HTML5 всё оформление текста производится в каскадной таблице стилей (CSS), в том числе и выравнивание текста на странице. Точно также выравнивание задаётся и заголовкам (h1-h6). Но про стили мы поговорим подробнее в следующих уроках.

HTML ссылки

Ссылки HTML определяются с помощью тега <a>:

Пример 6.6:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Ссылки в HTML</title>
</head>
<body>
<p>Это ссылка на сайт <a href="https://www.w3schools.com">w3schools.com</a></p>
</body>
</html>
Посмотреть результат!

Пункт назначения ссылки указан в атрибуте href. Атрибуты используются для предоставления дополнительной информации об элементах HTML.

HTML изображения

Изображения HTML определяются тегом <img>.

Исходный файл (src), альтернативный текст (alt), ширина и высота предоставляются в качестве атрибутов:

Пример 6.7:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Изображения в HTML</title>
</head>
<body>
<img src="https://www.w3schools.com/html/w3schools.jpg" alt="W3Schools.com" width="104" height="142">
</body>
</html>
Посмотреть результат!

Согласно спецификации HTML5 атрибут alt является ОБЯЗАТЕЛЬНЫМ для любого изображения на веб-странице!
