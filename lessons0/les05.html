Семантическая разметка веб-страницы

Приветствую вас на пятом уроке по HTML для начинающих, на котором вы узнаете, что такое семантическая разметка веб-страницы, какие есть семантические элементы в HTML5, а также что такое валидаторы кода и как проверить валидность веб-страницы.

Помимо логической структуры веб-страницы в спецификации HTML5 была введена семантическая разметка.

Семантика - это изучение значений слов и фраз. Семантические элементы - это элементы со значением.

Что такое семантические элементы на веб-странице? Какие они бывают?

Семантический элемент четко описывает его значение как для браузера, так и для разработчика. Примеры не семантических элементов: <div> и <span> - эти теги ничего не сообщают о своём содержании. Примеры семантических элементов: <form> (форма), <table> (таблица) и <article> (статья) - эти теги четко определяют своё содержимое.

Пример:

Теги <i></i> и <em></em> будут выглядеть на странице совершенно одинаково, то есть представлять фактически будут одно и то же, а семантически первый тег — это начертание курсивом, а второй — логическое выделение (браузеры выводят курсивом).

Семантические элементы HTML5 поддерживаются во всех современных браузерах.

Новые семантические элементы в HTML5

Многие веб-сайты содержат HTML-код, например: <div id = "nav"> <div class = "header"> <div id = "footer"> для указания навигации, заголовка и нижнего колонтитула. Но это так было раньше - до появления спецификации HTML5. Я, когда вижу на некоторых сайтах такую разметку, называю её дивная болезнь. Потому как количество тегов div просто зашкаливает! И очень усложняет HTML-код.

Сейчас HTML5 предлагает новые семантические элементы для определения различных частей веб-страницы:

  • <article> (статья)
  • <aside> (сторона)
  • <details> (детали, подробности)
  • <figcaption> (подпись под изображением)
  • <figure> (фигура, рисунок, цифра)
  • <footer> (футер, подвал, нижний колонтитул)
  • <header> (заголовок, верхний колонтитул)
  • <main> (главная, основное содержимое)
  • <mark> (метка)
  • <nav> (навигация)
  • <section> (раздел)
  • <summary> (резюме, краткий отчет)
  • <time> (время)

До появления стандарта HTML5 вся разметка страниц осуществлялась преимущественно с помощью элементов <div>, которым присваивали классы class или идентификаторы id для наглядности разметки (например, <div id="header">). С их помощью в HTML-документе размещали верхние и нижние колонтитулы, боковые панели, навигацию и многое другое.

Стандарт HTML5 предоставил новые элементы для структурирования, группировки контента и разметки текстового содержимого. Новые семантические элементы позволили улучшить структуру веб-страницы, добавив смысловое значение заключенному в них содержимому (было <div id="header">, стало <header>). Для отображения внешнего вида элементов не задано никаких правил, поэтому элементы можно стилизовать по своему усмотрению. Для всех элементов доступны ‎глобальные атрибуты.

  • Мета содержимое
  • Потоковое содержимое
  • Секционное содержимое
  • Заголовочное содержимое
  • Текстовое содержимое
  • Встроенное содержимое
  • Интерактивное содержимое

Глобальные атрибуты — это атрибуты общие для всех HTML элементов; они могут использоваться всеми элементами, хотя на некоторые элементы могут не оказывать влияния.

Подробнее глобальные атрибуты рассмотрим в следующих уроках.

А сейчас рассмотрим каждый из семантических элементов согласно спецификации HTML5.

HTML5 элемент <header>

Категории контента: потоковое содержимое.

Элемент <header> указывает заголовок для документа или раздела. Элемент <header> используется как контейнер для вводного содержимого, не является обязательным. Может содержать заголовки, оборачивать содержание раздела страницы, форму поиска или логотип. Может быть несколько элементов <header> в одном веб-документе и они могут располагаться в любой части страницы.

Пример 5.1: Здесь тег header используется в качестве обёртки для шапки страницы, внутри которой могут располагаться логотип и название сайта.

Пример 5.2: Здесь тег header используется в качестве заголовка для статьи (внутри article):

Элемент <header> нельзя помещать внутрь элементов <footer>, <address> или другого элемента <header>.

HTML5 элемент <section>

Элемент <section> определяет раздел в документе. Согласно документации W3C HTML5: «Раздел представляет собой тематическую группировку контента, как правило, с заголовком». Домашняя страница обычно может быть разделена на разделы для введения, содержания и контактной информации.

Категории контента: потоковое содержимое, секционное содержимое.

Элемент представляет собой универсальный раздел документа. Группирует тематическое содержимое и обычно содержит заголовок. Не является блоком-оберткой, для этих целей уместнее использовать элемент <div>. В качестве содержимого может выступать оглавление, разделы научных публикаций, программа мероприятия. Домашняя страница сайта также может быть поделена на секции — блок вводной информации, новости и контакты. Элемент рекомендуется использовать только в том случае, если содержимое элемента будет явно указано в схеме документа.

Пример 5.3: Две независимых секции из заголовками и текстом. 1-я секция - Цитаты Ницше, 2-я секция - Цитаты Мураками.

Пример 5.4: Статья (article) имеет заголовок, а также состоящая из двух секций, в каждой их которых есть свой заголовок и текст. Здесь заголовок статьи "Новогодние песни", затем 1-я секция из заголовком "Ёлочка", 2-я секция из заголовком "Зима в Простоквашино".

Как вы, наверное, заметили, при написании кода я уже не использую теги головы страницы <head>. Ведь согласно спецификации HTML5 их можно не использовать. Но при этом обязательным остаётся тег title (заглавия страницы). Также для правильного отображения текста не забывайте указывать язык страницы (в данном случае русский - ru) и кодировку utf-8.

HTML5 элемент <article>

Элемент <article> указывает независимый, автономный контент. Статья (article) должна иметь смысл сама по себе, и ее следует читать независимо от остальной части веб-сайта.

Категории контента: потоковое содержимое, секционное содержимое. Используется для группировки записей — публикаций, статей, записей блога, комментариев. Представляет собой независимый обособленный блок, предназначенный для многократного использования, как правило, начинается с заголовка. Может дублироваться на других страницах сайта и содержать внутри другие элементы <article>, которые по содержанию имеют близкое отношение к содержанию внешней статьи. Если на странице присутствует только одна статья с заголовком и текстовым содержимым, она не нуждается в обёртке элементом <article>. Элемент рекомендуется использовать только в том случае, если содержимое элемента будет явно указано в схеме документа.

Примеры использования элемента <article>:

  • Сообщение форума
  • Сообщение блога
  • Газетная статья

Пример 5.5: Статья (article) состоит из заголовка (между тегами h2) и двух абзацев (параграфов).

Как видно из данного примера, внутри одной статьи могут быть также свой заголовок (header) и подвал (footer), при этом заголовок, как правило, еще оборачивается тегами h1-h6, а внутри footer могут находиться комментарии к данной статье, ссылки на другие разделы сайта и т.д.

Вложенность <article> в <section> или наоборот

Элемент <article> указывает независимый, автономный контент. Элемент <section> определяет раздел в документе. При этом как элемент <article> может быть вложен в элемент <section>, так и наоборот, элемент <article> может содержать элемент <section>.

В Интернете можно найти HTML-страницы с элементами <section>, содержащими элементы <article>, и <article>, содержащими элементы <section>. Вы также найдете страницы с элементами <section>, содержащими элементы <section>, и <article> элементы, содержащие элементы <article>.

Пример для газеты: спортивные статьи в спортивной секции, возможно, имеют технический раздел в каждой статье.

Можно создавать родительские элементы <section> с вложенными элементами <article>, в которых есть один или несколько элементов <article>. Не все страницы должны быть устроены именно так, но это допустимый способ вложения элементов. Например, основная область контента страницы содержит два блока со статьями разной тематики. Можно сделать на этом акцент, поместив каждую статью одной тематики внутрь элемента <section>

Пример 5.7:

<!DOCTYPE html>
<html lang="uk">
<meta charset="utf-8">
<title>Елементы article и section. Вложенность элементов</title>
<body>
<section>
<h2>Раздел Frontend</h2>
<article>
<h3>Язык гипертекстовой разметки HTML</h3>
<p>HTML (от англ. HyperText Markup Language — «язык гипертекстовой разметки») — стандартизированный язык разметки документов во Всемирной паутине.</p>
</article>
<article>
<h3>Каскадные таблицы стилей CSS</h3>
<p>CSS (от англ. Cascading Style Sheets — каскадные таблицы стилей) — формальный язык описания внешнего вида документа, написанного с использованием языка разметки.</p>
</article>
</section>
<section>
<h2>Раздел Backend</h2>
<article>
<h3>Язык программирования PHP</h3>
<p>PHP (рекурсивный акроним словосочетания PHP: Hypertext Preprocessor) - это распространенный язык программирования общего назначения с открытым исходным кодом. PHP сконструирован специально для ведения Web-разработок и его код может внедряться непосредственно в HTML.</p>
</article>
<article>
<h3>Язык программирования SQL</h3>
<p>SQL - (англ. structured query language — «язык структурированных запросов») — язык программирования, применяемый для создания, модификации и управления данными в реляционной базе данных, управляемой соответствующей системой управления базами данных.</p>
</article>
</section>
</body>
</html>

HTML5 элемент <nav>

Элемент <nav> определяет набор навигационных ссылок.

Обратите внимание, что НЕ все ссылки документа должны находиться внутри элемента <nav>. Элемент <nav> предназначен только для основного блока ссылок навигации.

Категории контента: потоковое содержимое, секционное содержимое.

Предназначен для создания блока навигации веб-страницы или всего веб-сайта, при этом не обязательно должен находиться внутри <header>. На странице может быть несколько элементов <nav>. Не заменяет теги <ul> или <оl>, он просто их обрамляет. Не все группы ссылок на странице должны быть обёрнуты <nav>, этот элемент предназначен в первую очередь для разделов, которые состоят из главных навигационных блоков.


Пример 5.8: навигация в виде списка

<h3>Главное меню</h3>
<nav>
<ul>
<li><a href="#">Страница 01</a></li>
<li><a href="#">Страница 02</a></li>
<li><a href="#">Страница 03</a></li>
</ul>
</nav>

HTML5 элемент <footer>

Категории контента: потоковое содержимое. Представляет собой нижний колонтитул содержащей его секции или корневого элемента. Обычно содержит информацию об авторе статьи, данные об авторском праве (копирайте) и т.д. Если используется как колонтитул всей страницы, содержимое дополняется сведениями об авторских правах, ссылками на условия использования, контактную информацию, ссылками на связанное содержимое и т.п.

В одном веб-документе может быть несколько элементов <footer>. Как каждая страница, так и каждая статья может иметь свой элемент <footer>, более того, <footer> можно поместить в элемент <blockquote>, чтобы указать источник цитирования.

Некоторые варианты использования нижнего колонтитула (footer) вы уже могли видеть в предыдущих примерах.

Пример 5.10: Использование footer в качестве нижнего колонтитула всей страницы:

<body>
<header>Блок Шапка сайта</header>
<article>Блок Содержание сайта (статья)</article>
<footer>
copyright admin site © 2017. All Rights Reserved  
<a href="mailto:admin@gmail.com"> Связь с админом </a>
</footer>
</body>

HTML5 элемент <aside>

Иногда на странице размещается информация, дополняющая основной контент, - выносные цитаты, диаграммы, дополнительные мысли или сопутствующие ссылки. Для пометки таких элементов может использоваться новый тег aside.

Категории контента: потоковое содержимое, секционное содержимое.

Группирует содержимое, связанное с окружающим его контентом напрямую, но которое можно воспринимать отдельно. Удаление этого блока не повлияет на понимание основного содержимого. Обычно это какая-то сноска (как в книгах). Чаще всего элемент позиционируется как боковая колонка и включает в себя группу элементов: навигацию, цифровые данные, цитаты, рекламные блоки, архивные записи и т.п. Не подходит для блоков, просто позиционированных в стороне.

Пример 5.12:

<section>
<article>
<header>
<h2>Цитаты Стива Джобса</h2>
</header>
<aside>
<p> <i>Пройденный путь и есть награда.</i><br>(Стив Джобс) </p>
</aside>
<p>С годами я перестал покупать многие вещи просто потому, что они теперь кажутся мне нелепыми. (Стив Джобс)</p>
<p>Когда мне исполнилось 23 года, у меня уже было больше миллиона долларов, в 24 года — больше 10 миллионов, а в 25 лет — больше 100 миллионов. И всё это не имеет никакого значения, потому что я никогда ничего не делал просто ради денег. (Стив Джобс)</p>
<footer>
<p><a href="comments"><i>Комментарии</i></a>...</p>
</footer>
</article>
</section>

Размещение элемента aside может быть где угодно от основного текста. Само расположение элемента производится обычно с использованием стилей (CSS).

HTML5 элементы <figure> и <figcaption>

Категории контента: потоковое содержимое, корневое секционное содержимое.

Элемент <figure> представляет автономный контент (необязательно с заголовком), являющийся самостоятельным элементом основного потока. Элемент может быть перемещен из основного содержимого документа в боковую колонку или приложение, не затрагивая поток документа. С помощью элемента <figure> можно добавлять краткие характеристики к иллюстрациям, фотографиям, диаграммам, фрагментам кода и т.д., т.е. просто подписывать изображения.

Пример 5.13: Изображение имеет подпись.

<figure>
<img src="../img/dnipro.jpg" alt="Река Днепр" width="240" height="160">
<figcaption>Рис. 1. - Река Днепр. Украина</figcaption>
</figure>
Посмотреть результат!

Элемент <figure> является блочным, по ширине занимает всю ширину блока-контейнера за минусом внешних отступов margin.

Элемент <figcaption> — потомок элемента <figure>, не принадлежит ни к одной категории контента. Элемент является блочным, по ширине равен ширине элемента <figure>, высота по умолчанию равна 18px.

Элемент <img> определяет образ, элемент <figcaption> определяет подпись.

HTML5 элемент <main>

Категории контента: потоковое содержимое. Элемент <main> представляет основное содержимое документа (содержимое элемента <body>). Контент, находящийся внутри элемента, должен быть уникальным и не повторяться во всех документах сайта, таких как навигационные ссылки, информация о копирайте, логотипы, формы поиска (в случае, если форма поиска является основной функцией документа).

Элемент <main> не может быть потомком таких элементов как <article>, <aside>, <footer>, <header> или <nav>.

Пример 5.14:

<body>
<header><h3>Шапка сайта</h3></header>
<section><nav>Меню сайта</nav></section>
<main>
<h2>Основное содержимое</h2> <article>Статья</article>
</main>
<footer>Подвал сайта</footer>
</body>
Посмотреть результат!
HTML5 элемент <address>

Категории контента: потоковое содержимое. Используется для определения контактной информации автора/владельца документа или статьи. Для обозначения автора документа тег размещают внутри элемента <body>, для отображения автора статьи — внутри тега <article>. В браузере обычно отображается курсивом. Большинство браузеров будут добавлять разрыв строки до и после элемента <address> и отобразит элемент <address> со следующими значениями по умолчанию:

address {
display: block;
font-style: italic;
}

Пример 5.15:

<body>
<article>
<h3>Заголовок</h3>
<p>Текст</p>
<address>Автор статьи</address>
</article>
</body>

  • Элемент <address> обычно включается вместе с другой информацией в элемент <footer>.
  • Тег <address> также поддерживает глобальные атрибуты в HTML.
  • Тег <address> также поддерживает атрибуты событий в HTML.
HTML5 элемент <time>

Категории контента: потоковое содержимое, текстовое содержимое. Определяет время (24 часа) или дату по григорианскому календарю с возможным указанием времени и смещения часового пояса. Текст, заключенный в данный тег, не имеет стилевого оформления браузером. Для тега доступен атрибут datetime, в качестве содержимого которого указывается то, что будет видеть пользователь на экране своего компьютера:

<p>Я праздную дату всех влюблённых <time datetime="2018-02-14 20:00">14 февраля - День Святого Валентина</time>.</p>

Результат:

Я праздную дату всех влюблённых 14 февраля - День Святого Валентина.


Элемент time не отображает ничего особенного в любом из основных браузеров, кроме самой даты, которую вы укажете.

Этот элемент также может использоваться для кодирования дат и времени машиносчитываемым способом, чтобы user-агенты могли предлагать добавление напоминания дня рождения или запланированные события в календарь пользователя, а поисковые системы могли выводить более интеллектуальные результаты поиска.

Тег time поддерживается всеми основными браузерами. В таблице указаны браузеры, начиная с версии, которая полностью поддерживает данный элемент.

HTML5 элемент <mark>

Категории контента: потоковое содержимое, текстовое содержимое. Текст, помещенный внутрь тега <mark>, выделяется по умолчанию желтым цветом (цвет фона и цвет шрифта в выделенном блоке можно изменить, задав определенные css-стили). С помощью данного тега можно отмечать важное содержимое, а также ключевые слова.

Пример:

<p>HTML5 (англ. <mark>HyperText Markup Language</mark>, version 5) — язык для структурирования и представления содержимого всемирной паутины.</p>

Результат:

HTML5 (англ. HyperText Markup Language, version 5) — язык для структурирования и представления содержимого всемирной паутины.

Тег mark поддерживается всеми основными браузерами. В таблице указаны браузеры, начиная с версии, которая полностью поддерживает данный элемент.

HTML5 элементы <details> и <summary>

Тег <details> указывает дополнительные данные, которые пользователь может просматривать или скрывать по требованию.

Тег <details> можно использовать для создания интерактивного виджета, который пользователь может открыть и закрыть. Любой контент можно поместить в тег <details>.

Содержимое элемента <details> не должно быть видимым, если не установлен атрибут open.

Тег <details> может иметь атрибут open, который указывает, что детали должны быть видимыми (открытыми) для пользователя.

Тег <details> также поддерживает глобальные атрибуты в HTML.

Тег <details> также поддерживает атрибуты событий в HTML.

Тег details поддерживается не всеми браузерами. В таблице указаны браузеры, начиная с версии, которая полностью поддерживает данный элемент.

Тег <summary> используется для указания видимого заголовка для <details>. Заголовок можно кликнуть, чтобы просмотреть / скрыть данные.

Элемент <summary> должен быть первым дочерним элементом элемента <details>

Тег <summary> также поддерживает глобальные атрибуты в HTML.

Тег <summary> также поддерживает атрибуты событий в HTML.

<p>Текст статьи</p>
<details>
<summary>Заголовок для скрытого текста. Нажмите здесь!</summary>Скрытый текст статьи.
</details>
</article>

Валидаторы кода. Проверка валидности веб-страниц

А теперь, после того, как мы разобрались с семантикой веб-страницы и написанием кода согласно спецификации HTML5, давайте проверим валидность кода любой из наших страниц согласно данной спецификации.

Валидность - это написание HTML-кода в соответствии с определенными стандартами, разработанными Консорциумом Всемирной Паутины - (W3C).

Для проверки веб-страниц на наличие ошибок и замечаний существует множество путей и способов. Условно они делятся на онлайновые и локальные. Онлайновые предназначены для проверки страниц с помощью браузера через Интернет, а локальные используются для проверки документов на текущем компьютере. Давайте рассмотрим популярные методы валидации документов.

По адресу https://validator.w3.org/ располагается, пожалуй, самый распространённый инструмент для проверки отдельных страниц на валидность. Этот сайт предлагает три способа проверки: по адресу, локального файла и введенного в форму кода.

Проверка по адресу

Если ваш сайт уже опубликован в Интернете, то любую страницу можно проверить, вводя в текстовое поле её адрес. Вводя url-адрес сайта в форме (валидация по адресу) и нажав кнопку Check (проверить) получим сообщение о том, валидный документ или нет. При этом проверяется, естественно не весь сайт, а только страница, адрес которой вы указали. Валидатор проверяет HTML-код страницы и в случае отсутствия ошибок докладывает о валидности документа. При обнаружении ошибок выводится уведомление о том, что страница не валидна и список ошибок с указанием строк, где встречаются ошибки.

Проверка локальных файлов

Документы, еще не выставленные в Интернете, можно проверить с помощью формы, озаглавленной «Validate by File Upload» (валидация загруженных файлов). Вначале следует указать путь к HTML-файлу, после чего нажать кнопку Check. Файл будет загружен на сервер и проверен на ошибки.

Использование формы для ввода кода

В некоторых случаях требуется проверить код без сохранения его в отдельный файл. В этом случае вы просто копируете и вставляете HTML-код в форме для текста и отправляете его на сервер для валидации.

Еще один альтернативный валидатор HTML5 от сайта WHATWG checker.html5.org. Проверка веб-страниц на валидность проводится аналогичным образом.

Также в некоторых браузерах (например, в Mozilla Firefox) есть возможность установить дополнительные расширения, в т.ч. и для валидации документа прямо в браузере. В этом случае вам даже не понадобится Интернет, вы сможете проверить HTML-код в самом браузере.


Почему необходимо использовать семантические элементы?

С HTML4 разработчики использовали свои собственные имена id/class для элементов стиля: header, top, bottom, footer, menu, navigation, main, container, content, article, sidebar, topnav и т.д.

Это не позволяло поисковым системам определять правильное содержимое веб-страницы.

С новыми элементами HTML5 (<header> <footer> <nav> <section> <article>) это стало проще.

Согласно заявлению W3C, семантические веб-элементы: «Позволяют делиться и повторно использовать данные для всех приложений, организаций и сообществ».
