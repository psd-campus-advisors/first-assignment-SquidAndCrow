HTML. Элементы

Элементы HTML

Как вы уже знаете из предыдущих уроков, HTML-элемент обычно состоит из начального тега и конечного тега с содержимым, вставленным между тегами:

<название тега> Здесь идёт содержимое... </ название тега>

Т.е. элемент HTML - это всё: от начального тега до конечного тега:

<p>Мой первый абзац.</p>
Начальный тег Содержание элемента Конечный тег
<h1> Первый заголовок </h1>
<p> Первый параграф </p>
<br> Разрыв строки Одиночный тег

Элементы HTML без содержимого называются пустыми элементами. Пустые элементы не имеют конечного тега, например, элемент, который указывает на разрыв строки <br>.

Вложенные элементы HTML

HTML-элементы могут быть вложенными (элементы могут содержать элементы).

Все HTML-документы состоят из вложенных элементов HTML.

Этот пример 7.1 содержит несколько элементов HTML:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Вложенные элементы</title>
</head>
<body>
<h1>Первый заголовок</h1>
<p>Первый параграф.</p>
</body>
</html>
Посмотреть результат!

Пример и разъяснение

Элемент <html> оборачивает весь документ. У него есть начальный тег <html> и конечный тег </html>. HTML-документ содержит два основных элемента, которые оборачивают другие элементы: <head> и <body>. Элемент <head> содержит в себе элементы <meta> и <title>. Непосредственно на веб-странице они не отображаются. Содержимое, которое отображается на странице, находится внутри элемента <body>.

<html>
<body>
<h1>Первый заголовок</h1>
<p>Первый параграф.</p>
</body>
</html>

Элемент <body> оборачивает тело документа. У него есть начальный тег <body> и конечный тег </body>. Содержимое элемента <body> - это два других элемента HTML (<h1> - заголовок и <p> - параграф).

<body>
<h1>Первый заголовок<h1>
<p>Первый параграф.</p>
</body>

Элемент <h1> оборачивает заголовок. Он имеет начальный тег <h1> и конечный тег </h1>. Содержимое элемента: Первый заголовок.

<h1>Первый Заголовок</h1>

Элемент <p> определяет параграф. Он имеет начальный тег <p> и конечный тег </p>. Содержимое элемента: Первый абзац.

<p>Первый параграф.</p>
Не забывайте о конечном теге

Некоторые элементы HTML будут отображаться правильно, даже если вы забудете конечный тег:

Пример 7.2:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Если отсутствует конечный тег</title>
</head>
<body>
<p>Это параграф
<p>Это параграф
</body>
</html>
Посмотреть результат!

Приведенный выше пример работает во всех браузерах, поскольку закрывающий тег считается необязательным.

НО никогда не полагайтесь на это! Т.к. это может привести к неожиданным результатам и / или ошибкам, если вы забудете конечный тег.

Пустые элементы HTML

Элементы HTML без содержимого называются пустыми элементами.

<br> - пустой элемент без закрывающего тега (тэг <br> определяет разрыв строки).

Пустые элементы могут быть «закрыты» в открывающемся теге следующим образом: <br/>.

HTML5 не требует, чтобы пустые элементы были закрыты. Но если вы хотите получить более строгую проверку, или если вам нужно сделать ваш документ доступным для чтения парсерами XML, вы должны закрыть все элементы HTML правильно.

Использование тегов в нижнем регистре

HTML-теги не чувствительны к регистру: <P> означает то же, что и <p>. Стандарт HTML5 не требует строчных тегов, но W3C рекомендует использовать строчные буквы в HTML и требует использовать строчные буквы для более строгих типов документов, таких как XHTML

W3C РЕКОМЕНДУЕТ использовать строчные теги (т.е. в нижнем регистре).

HTML. Атрибуты

Атрибуты предоставляют дополнительную информацию об элементах HTML.

Атрибуты HTML

  • Все элементы HTML могут иметь атрибуты
  • Атрибуты предоставляют дополнительную информацию об элементе
  • Атрибуты всегда указываются в начальном теге
  • Атрибуты обычно входят в пары имя / значение, например: name = "value"

Атрибут href

HTML-ссылки определяются тегом <a>. Адрес ссылки указан в атрибуте href:

<a href="https://www.w3schools.com">Это ссылка</a>

Атрибут src

Изображения HTML определяются тегом <img>.

Имя файла источника изображения указано в атрибуте src:

<img src="img_girl.jpg">

Атрибуты ширины (width) и высоты (height)

Изображения в HTML имеют набор атрибутов размера, который определяет ширину и высоту изображения:

<img src="img_girl.jpg" width="500" height="600">

Размер изображения задается в пикселях: width = "500" означает ширину 500 пикселей, height = "600" означает высоту в 600 пикселей.

Атрибут alt

Атрибут alt указывает альтернативный текст, который будет использоваться, когда изображение невозможно отобразить (если оно отсутствует). Значение атрибута можно считывать с экрана. Допустим, слабовидящий или слепой человек «слушает» веб-страницу (с помощью аудио), и он может «услышать» элемент, т.е. тот текст, что будет написан в атрибуте alt.

<img src="girl_in_dress.jpg" alt="Девушка в платье">

Атрибут alt также полезен, если изображение не существует. Что произойдет, если мы попытаемся отобразить изображение, которое не существует:

<img src="girl_is_naked.jpg" alt="Девушка голая">

Если мы попытаемся отобразить изображение, которое не существует, вместо него будет отображаться значение атрибута alt.

Атрибут style

Атрибут style используется для указания стиля элемента, например цвета, шрифта, размера и т.д.

<p style="color:red">Это параграф красного цвета</p>

Используйте атрибут style для оформления стиля элемента.

Атрибут lang

Язык документа может быть объявлен в теге <html>. Язык объявлен с атрибутом lang. Объявление языка важно для приложений доступа (программы для чтения с экрана) и поисковых систем:

<!DOCTYPE html> <html lang="en-US"> <body>Тело страницы</body> </html>

Первые две буквы указывают язык (en). Если есть диалект, используйте ещё две буквы, например en-US (американский английский).

Атрибут title

Здесь атрибут title добавляется к элементу <p>. Значение атрибута title будет отображаться в виде подсказки при наведении указателя мыши на абзац.

Не путайте атрибут title с тегом <title>, который размещается в разделе <head>!

Пример 7.3: Всплывающая подсказка с помощью атрибута title:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Атрибут title</title>
</head>
<body>
<h2 title="Я - заголовок">Атрибут title</h2>
<p title="Я - подсказка">Это первый параграф.</p>
Наведите мышку над первым параграфом, чтобы отобразить атрибут title как всплывающую подсказку. Данный атрибут, естественно, не работает в тачскринах!</p>
</body>
</html>
Посмотреть результат!

Рекомендуется использовать строчные атрибуты!

Стандарт HTML5 не требует строчных имен атрибутов. Атрибут title может быть записан с прописными или строчными буквами, такими как title или TITLE. W3C рекомендует строчные буквы в HTML и требует строчных букв для более строгих типов документов, таких как XHTML.

В W3C всегда используются имена атрибутов нижнего регистра.

Предлагается использовать кавычки вокруг значения атрибута

Стандарт HTML5 не требует кавычек вокруг значений атрибутов. Атрибут href, продемонстрированный выше, может быть записан как:

<a href=https://www.w3schools.com> Это ссылка</a>

W3C рекомендует кавычки в HTML и требует кавычки для более строгих типов документов, таких как XHTML. Иногда необходимо использовать кавычки. В этом примере не будет отображаться атрибут title правильно, поскольку он содержит пробел:

<p title=About W3Schools>

Вы не можете пропустить кавычки вокруг значения атрибута, если значение содержит пробелы.

Если вы переместите указатель мыши над абзацем выше, ваш браузер отобразит только первое слово из заголовка.

Использование кавычек является наиболее распространенным. Опускание кавычек может привести к ошибкам. В W3C всегда используются кавычки вокруг значений атрибутов.

Одиночные или двойные кавычки?

Двойные кавычки вокруг значений атрибутов являются наиболее распространенными в HTML, но можно использовать и одинарные кавычки. В некоторых ситуациях, когда само значение атрибута содержит двойные кавычки, необходимо использовать одинарные кавычки:

<p title='John "ShotGun" Nelson'>

Или наоборот:

<p title="John 'ShotGun' Nelson">

Краткое содержание раздела Атрибуты:

  • Все элементы HTML могут иметь атрибуты
  • Атрибут title предоставляет дополнительную информацию об инструменте
  • Атрибут href предоставляет адресную информацию для ссылок
  • Атрибуты width и height предоставляют информацию о размере изображений
  • Атрибут alt предоставляет текст для чтения с экрана
  • В W3Schools всегда используются строчные имена атрибутов
  • В W3Schools всегда указываются значения атрибутов с двойными кавычками

HTML. Заголовки

Заголовки определяются тегами c <h1> до <h6>. <h1> определяет наиболее важный заголовок (самого большого размера). <h6> определяет наименее важный заголовок (самого маленького размера).

Как в приведённом примере:

<h1>Заголовок 1</h1>

<h2>Заголовок 2</h2>

<h3>Заголовок 3</h3>

<h4>Заголовок 4</h4>

<h5>Заголовок 5</h5>

<h6>Заголовок 6</h6>

Примечание. Браузеры автоматически добавляют некоторое свободное пространство (отступ) до и после заголовка.

Заголовки важны

Поисковые системы используют заголовки для индексации структуры и содержимого веб-страниц. Пользователи просматривают страницы по заголовкам. Для отображения структуры документа важно использовать заголовки. Заголовки <h1> должны использоваться для основных заголовков, затем <h2> заголовки, затем менее важные <h3> и т.д.

Для соответствия спецификации HTML5 требуется, чтобы самый большой заголовок <h1> использовался на веб-странице только один раз, например, в названии сайта (в логотипе). Все остальные заголовки с <h2> по <h6> могут использоваться на странице многократно.

Примечание. Используйте заголовки HTML только для заголовков. Не используйте заголовки для создания БОЛЬШОГО текста или просто текста жирным шрифтом.

Большие заголовки

Каждый заголовок HTML имеет размер по умолчанию. Однако вы можете указать размер для любого заголовка с атрибутом style:

<h1 style="font-size:60px;">Heading 1</h1>

Также вы можете указать размеры заголовков (с <h1> по <h6>) по умолчанию в отдельном файле CSS (каскадной таблице стилей), которую мы будем рассматривать чуть позже в следующих уроках.

В спецификации HTML5 существует всего 6 различных размеров заголовков, начиная с <h1> до <h6>, которые отличаются по размерам. Но в качестве эксперимента давайте создадим еще один заголовок <h7>.

Пример 7.4:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Еще один заголовок h7</title>
</head>
<body>
<h7>Здесь заголовок h7</h7>
<p>Здесь параграф</p>
</body>
</html>
Посмотреть результат!

Посмотрим результат! И мы видим, что заголовок <h7>, которого вообще не существует в HTML, работает как обычный параграф, т.е. текст отображается точно также, как заключенный между тегами <p></p>. Но если мы зададим этому заголовку стиль (можно в самом файле HTML, а можно в отдельном файле CSS), допустим, присвоим ему размер в 0.5em (т.е. половина от размера шрифта по умолчанию), то данный заголовок, заключенный в теги <h7></h7> приобретёт эти свойства.

Пример 7.5:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Еще один заголовок h7</title>
</head>
<body>
<h7 style="font-size:0.5em">Здесь заголовок h7</h7>
<p>Здесь параграф</p>
</body>
</html>
Посмотреть результат!

Но это я показываю всего лишь как эксперимент. Использовать же на практике несуществующий в спецификации HTML5 тег h7 всё же не стоит.

Горизонтальные разделители

Тег <hr> (horisontal rule - горизонтальный разделитель, линейка) определяет тематический разрыв на странице HTML и чаще всего отображается как горизонтальная линия. Элемент <hr> используется для разделения содержимого (или определения изменения) на HTML странице:

Пример 7.6:

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Горизонтальный разделитель hr</title>
</head>
<body>
<h1>Это заголовок 1</h1>
<p>Это какой-то текст.</p>
<hr>
<h2>Это заголовок 2</h2>
<p>Это какой-то другой текст.</p>
<hr>
</body>
</html>
Посмотреть результат!

Как просмотреть исходный HTML-код страницы?

Вы когда-нибудь видели веб-страницу и задавались вопросом: «Интересно, а как они это сделали?»

Просмотреть исходный код HTML:

Чтобы просмотреть исходный код HTML-страницы, кликните правой кнопкой мыши на странице и выберите в контекстном меню «View Page Source» / «Программный код страницы» (в Firefox) или «Просмотреть источник страницы» (в Chrome) или аналогичный в других браузерах. Откроется окно, содержащее исходный HTML-код страницы.

Просмотреть элемент HTML:

Кликните правой кнопкой мыши элемент (или пустую область) и выберите «Inspect» / «Исследовать» или «Inspect Element» / «Проверить элемент», чтобы увидеть, из каких элементов состоит страница (вы увидите как HTML-код, так и CSS). Вы также можете редактировать HTML или CSS на лету в раскрывающемся меню Элементы или стили.

Тег Описание
<html> Определяет корень HTML-документа
<body> Определяет тело документа
<head> Контейнер для всех элементов заголовка (название, скрипты, стили, метаинформация и т.д.)
от <h1> до <h6> Определяет заголовки HTML
<hr> Определяет тематическое изменение содержимого

HTML. Параграфы (абзацы)

Элемент HTML <p> определяет параграф:

<p>Это параграф</p>
<p>Это другой параграф.</p>

Примечание. Браузеры автоматически добавляют некоторое свободное пространство (отступ) до и после параграфа.

Отображение HTML страниц на разных дисплеях

Вы не можете быть уверены, как будет отображаться HTML-документ на разных дисплеях. Большие или маленькие экраны и измененные окна покажут разные результаты. С помощью разметки HTML вы не можете изменить вывод на экран, добавив лишние пробелы или дополнительные строки в свой HTML-код. При отображении страницы браузер автоматически удалит лишние пробелы и дополнительные строки:

<p> Этот параграф содержит
много строк в исходном коде,
но браузер игнорирует их.</p>
<p>Этот параграф
содержит    много    пробелов
в исходном    коде,    
но браузер    игнорирует их.</p>

Не забывайте завершающий (конечный) тег!

Большинство браузеров отобразит HTML правильно, даже если вы забудете конечный тег:

<p>Это параграф.
<p>Это другой параграф.

Этот пример будет работать в большинстве браузеров, но не полагайтесь на него.

Примечание. Отбрасывание конечного тега может привести к неожиданным результатам или ошибкам, поэтому всё же лучше его использовать!

Разрыв строки в HTML

Элемент <br> в HTML определяет разрыв строки. Используйте его, если хотите разбить строку (или создать новую строку), не создавая новый абзац:

<p>Это<br>параграф<br>с разрывом строки.</p>

Тег <br> - одиночный, т.е. у него нет закрывающего тега.

Проблема стихотворения

Часто на странице необходимо использовать стихотворную форму, где каждая строка стихотворения должна отображаться с новой строки.

Это стихотворение будет отображаться на одной строке, даже если мы в редакторе будем писать как и положено для стиха - каждую новую строку с новой строчки:

Пример 7.7:

<!DOCTYPE html>
<html lang="uk">
<head>
<meta charset="UTF-8">
<title>Проблема віршів</title>
</head>
<body>
<p>
Любіть Україну, як сонце, любіть,

як вітер, і трави, і води…

В годину щасливу і в радості мить,

любіть у годину негоди.
</p>
</body>
</html>
Посмотреть результат!

Как же лучше форматировать стихотворения? Для этого есть несколько способов.

Элемент HTML <pre>

Элемент HTML <pre> определяет преформатированный текст. Текст внутри элемента <pre> отображается шрифтом фиксированной ширины (по умолчанию используется шрифт Courier), и он сохраняет как пробелы, так и разрывы строк:

Пример 7.8:

<!DOCTYPE html>
<html lang="uk">
<head>
<meta charset="UTF-8">
<title>Елемент PRE</title>
</head>
<body>
<pre>
Любіть Україну у сні й наяву,
вишневу свою Україну,
красу її, вічно живу і нову,
і мову її солов'їну.
</pre>
</body>
</html>
Посмотреть результат!

Это самый простой и быстрый способ написания стихотворной формы - как текст набран в редакторе, так он и будет отображаться в браузере. Но у этого способа есть один недостаток - отсутствие адаптивности. Т.е. при изменении разрешения экрана текст будет неизменным, а значит, возможно появление горизонтальных полос прокрутки на странице, что не очень удобно для пользователей. Поэтому наиболее часто используется способ написания стихотворений с разрывом строки, т.е. с тегами <br> в конце каждой стихотворной строки. А тег <pre> чаще всего используется для отображения на странице, например, программного или html-кода, где необходимо учитывать правильное форматирование текста.
